# 【核心生存链路 · Golden Loop】

```text
[建立连接] -> [点对点发送纯文本] -> [服务端持久化并投递]
```

# 【V1 冻结决议】

- V1 仅实现：用户建立连接后，向指定对方ID发送纯文本（UTF-8 字符串）消息；服务端同步落库并向在线接收方转发（接收方不在线则仅存储）。
- V1 不做任何其他事。

# 【DEFERRED 清单】

- 手机号验证码登录、SMS 接入、JWT 鉴权、多端同步登录
- 图片/视频/语音消息、波形图、暂停播放、对象存储接入
- 阅后即焚（倒计时、本地彻底删除、服务端物理删除、防截屏/录屏、截屏通知、Key Escrow、6个月日志留存要求适配）
- 群聊（含500人大群）、群管理（踢人/禁言/公告）、消息扩散推拉结合与削峰方案、压测体系
- 动态墙（图文/短视频）、点赞、评论、可见性/隐私设置
- 支付集成（微信/支付宝/IAP）、红包（P2P/群拼手气）、任何资金流转与二清合规方案
- 实名制/KYC（二次实名认证、绑卡/身份证）
- 第三方内容安全API接入、UGC机审、举报与人工复审机制、群主责任制
- 服务端加密存储（Encryption at Rest）、端到端加密（E2EE）
- “永久云端存储”承诺、冷热分层/归档策略、上传压缩与单文件大小限制
- Telegram级性能目标（60/120fps）、Offline-first本地库优先与后台异步发送、渐进式图片加载与内存缓存
- 增量同步、换机拉取会话索引与按需拉历史消息
